## Qué es

Un artefacto de ingeniería concreto, no un SaaS. Las piezas entregadas:

- **Pipeline de dataset** — sintetizar NDAs, MSAs y licencias vía GPT-4o con enforcement estricto de JSON schema, luego curar (longitud, idioma, schema, dedup), dividir de forma estratificada con un guard de hash anti-contaminación, y formatear en JSONL de chat-template.
- **Entrenamiento** — QLoRA (NF4 + LoRA, rank 16, alpha 32, targets `all_linear`) sobre Llama 3.1 8B. Un único `TrainingConfig` compartido maneja Unsloth en la 4090 y el `SFTTrainer` de TRL en M1.
- **Eval** — tres predictores detrás de un `Protocol` (fixture, fine-tuned local, GPT-4o) corren sobre el mismo split de prueba reservado, reportando validez JSON, puntajes por campo, totales de tokens, latencia y costo de API.
- **Modelo de costo** — dólares de entrenamiento (horas de GPU, API de síntesis, API de eval) más una estimación self-hosted de $/1M tokens, reducida a un volumen de punto de equilibrio frente a seguir llamando a GPT-4o.
- **Publicación** — renderer de model card y subida del adaptador al Hugging Face Hub.
- **CI** — GitHub Actions corre Ruff, mypy (strict), pytest y commitlint en cada PR. Sin GPU en CI; la corrida paga es un paso manual, guiado por la metodología, en RunPod.

## Por qué lo construí

Para demostrar que puedo hacer fine-tuning y publicar un modelo abierto de punta a punta, con una historia de costos defendible, en lugar de solo llamar a una API hosteada. La pregunta interesante no es "¿puede QLoRA hacer fine-tuning de Llama 3.1 8B?" (puede), sino "¿el fine-tune realmente cierra la brecha con un modelo frontier en una tarea acotada, y cuánto tiempo tarda en amortizarse el costo único de entrenamiento?" Anvil está construido para responder eso con números que se remontan a inputs reales, y se complementa con Forge para que los dos cubran entrenamiento y serving como una sola historia de portfolio.

## Cómo funciona

### Síntesis de datos con un guard anti-contaminación

El set de entrenamiento son contratos sintéticos generados bajo un JSON schema estricto, por lo que cada muestra tiene una extracción ground-truth conocida. Antes del entrenamiento, un guard hashea el texto normalizado del contrato de cada muestra y aborta la corrida si algún documento aparece en más de un split. Eso elimina la forma más común en que un benchmark de fine-tuning infla silenciosamente sus propios puntajes.

### QLoRA en una sola 4090, ensayado en M1

El mismo orquestador que corre el job pago en RunPod corre en modo dry-run en una MacBook M1 base contra fixtures del repo, de punta a punta (`data-smoke → train-smoke → eval-smoke → cost`) con cero gasto en API. Ese ensayo debe pasar antes de rentar una GPU paga, así que los errores de config y los bugs de parser no cuestan nada en lugar de quemar minutos de 4090. La corrida completa tiene un tope de presupuesto cercano a $100, y la síntesis aborta en un tope fijo antes de cualquier gasto de entrenamiento.

### Eval triple condicionada a validez JSON

Los predictores base, fine-tuned y GPT-4o responden los mismos prompts. La validez es la condición: un output que no parsea contra el schema no recibe crédito por campo, porque un número F1 sobre JSON no parseable no tiene sentido. Se espera que el adaptador fine-tuned alcance una conformidad casi total con el schema donde el modelo base falla en la mayoría de los prompts, pero esas cifras permanecen marcadas como ilustrativas hasta que la corrida paga las mida.

### Costo y punto de equilibrio

El reporte de costos suma horas de GPU, API de síntesis y API de eval en un costo único de entrenamiento, luego calcula un $/1M tokens de inferencia self-hosted y el volumen al que el ahorro por token amortiza el entrenamiento frente a seguir llamando a GPT-4o. El throughput medido de Forge no se cita deliberadamente hasta que llegue la corrida paga de Forge; Anvil cita en cambio una fuente publicada de Llama 3.1 8B y 4090, para que ningún número dependa de una suposición no medida.

## Estado

El pipeline completo está listo y ensayado de punta a punta en M1 con cero gasto en API. La corrida paga en RunPod RTX 4090 que reemplaza cada cifra ilustrativa con una medida está en cola; cuando llegue, un solo commit intercambia los números reales de validez, por campo y de costo, y el writeup final sigue.
