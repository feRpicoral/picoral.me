## Qué hace

Un representante de ventas sube una grabación de llamada. Unos veinte segundos después, Sonar devuelve:

1. Investigación sobre la empresa del prospecto (Tavily + Claude Haiku 4.5)
2. Análisis estructurado de la llamada — temas, puntos de dolor, objeciones, tareas pendientes, sentimiento (Claude Sonnet 4.6)
3. Un próximo paso recomendado con puntos de conversación y nivel de urgencia (Claude Sonnet 4.6)
4. Un borrador de email de seguimiento con citas entre corchetes que enlazan a segmentos del transcript (Claude Sonnet 4.6)

El revisor ve una UI dividida: el email a la izquierda, el transcript a la derecha. Al pasar el cursor sobre una cita, se resalta el segmento correspondiente. El revisor puede aprobar, editar el cuerpo directamente o regenerar el writer con feedback. La regeneración reutiliza el estado previo de research, analysis y strategy; solo vuelve a correr el writer.

## Por qué lo construí

No es un producto — es un proyecto personal. Quería un solo codebase que ejercitara el stack moderno de IA de punta a punta: orquestación multi-agente con estado persistente, manejo real de audio, tool use estructurado, prompt caching y la forma multi-tenant B2B. Elegí la revisión de llamadas de ventas porque el flujo es genuinamente lineal con rollback, que es el punto ideal para grafos de agentes.

## Cómo funciona

### Orquestación multi-agente

- Cuatro nodos secuenciales (research, transcription, analysis, strategy, writer)
- Cada nodo devuelve output estructurado mediante Anthropic tool use + un schema Zod. Sin outputs de texto libre.
- Cada paso escribe una fila `AgentRunStep`. La ejecución se pausa en `AWAITING_APPROVAL` tras el paso del writer.
- Anthropic prompt caching habilitado en los mensajes de sistema — reducción de ~70% en tokens de entrada en ejecuciones repetidas.
- La regeneración del writer reutiliza el estado de los nodos anteriores.
- Ejecución en background mediante el route handler `after()` de Next.js 16 (`maxDuration = 300`).

### Procesamiento de audio

- La subida por drag-and-drop va directo del navegador a Supabase Storage mediante signed URL. El servidor no está en el camino de la subida.
- Groq Whisper Large v3 transcribe con timestamps a nivel de segmento.
- Tipo MIME y límite de 100 MB aplicados tanto en el server action como en la política del bucket.
- El writer recibe los segmentos del transcript etiquetados con índices entre corchetes; las citas referencian esos índices.
- La UI dividida hace scroll hasta el segmento citado al pasar el cursor.

### Multi-tenant B2B

Workspaces, roles (`OWNER`, `ADMIN`, `MEMBER`), invitaciones, API keys, log de auditoría. Tres capas de aislamiento — scoping por fila en Postgres, guards a nivel de aplicación en cada server action y políticas de bucket en Supabase Storage con clave por workspace ID.
