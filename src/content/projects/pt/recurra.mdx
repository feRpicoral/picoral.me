## O que faz

Recurra é uma plataforma de cobrança recorrente para negócios locais de serviços — negócios recorrentes, repetidamente. O proprietário gerencia serviços, agendas e clientes a partir de um painel admin em React ou de um app mobile em Expo. O cliente gerencia assinaturas de autoatendimento em um portal hospedado. Stripe cuida das cobranças. Uma integração com Clover POS feita em Kotlin permite checkout presencial que busca a assinatura ativa do cliente.

## Por que eu construí

Não é um produto — é um projeto pessoal. Eu queria construir algo que abrangesse web + mobile + POS + infraestrutura AWS em um único monorepo, em parte para aprender Nx e CDK de verdade, em parte porque a cobrança por assinatura para negócios locais de serviços é um domínio que acho genuinamente interessante (e consistentemente mal atendido).

## Como funciona

Monorepo Nx com workspaces NPM. Seis apps:

- `backend` — NestJS com Prisma contra PostgreSQL no RDS, implantado no App Runner.
- `web` — SPA React Router 7 em build/client/, hospedado em S3 + CloudFront.
- `mobile` — Expo.
- `clover-pos` — Android em Kotlin para Clover Mini.
- `infra` — AWS CDK com seis stacks: Network, Dns, Database, Backend, Web, Email.
- `backend-e2e` — testes e2e com Jest.

Tipos compartilhados e schemas Zod ficam em `@recurra/shared` e são importados por todos os apps. As stacks CDK usam SSM Parameter Store para segredos e um script de entrada que constrói a URL do banco de dados a partir do JSON do Secrets Manager.

## Status

Em desenvolvimento ativo. O backend, o admin web e a infra estão prontos; o app Clover POS e o app mobile do proprietário são os próximos.
