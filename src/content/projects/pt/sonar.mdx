## O que faz

Um rep de vendas faz upload de uma gravação de chamada. Cerca de vinte segundos depois, o Sonar retorna:

1. Pesquisa sobre a empresa do prospect (Tavily + Claude Haiku 4.5)
2. Análise estruturada da chamada — tópicos, dores, objeções, itens de ação, sentimento (Claude Sonnet 4.6)
3. Próximo passo recomendado com talking points e urgência (Claude Sonnet 4.6)
4. Rascunho de e-mail de follow-up com citações entre colchetes linkando de volta a trechos da transcrição (Claude Sonnet 4.6)

O revisor vê uma UI em split-view: e-mail à esquerda, transcrição à direita. Passar o mouse sobre uma citação destaca o trecho correspondente. O revisor pode aprovar, editar o corpo diretamente ou regerar o writer com feedback. A regeração reutiliza o estado anterior de pesquisa, análise e estratégia; só o writer roda novamente.

## Por que construí

Não é um produto — é um projeto pessoal. Queria uma codebase que exercitasse a stack moderna de IA de ponta a ponta: orquestração multi-agente com estado persistente, processamento real de áudio, tool use estruturado, prompt caching e o formato B2B multi-tenant. Revisão de chamadas de vendas foi o domínio escolhido porque o fluxo é genuinamente linear-com-rollback, que é o ponto ideal para grafos de agentes.

## Como funciona

### Orquestração multi-agente

- Quatro nós sequenciais (research, transcription, analysis, strategy, writer)
- Cada nó retorna output estruturado via tool use da Anthropic + um schema Zod. Sem outputs em texto livre.
- Cada etapa grava uma linha `AgentRunStep`. O run pausa em `AWAITING_APPROVAL` após o passo do writer.
- Prompt caching da Anthropic habilitado nas system messages — redução de ~70% nos tokens de entrada em runs repetidos.
- A regeração do writer reutiliza o estado dos nós anteriores.
- Execução em background via route handler `after()` do Next.js 16 (`maxDuration = 300`).

### Processamento de áudio

- Upload por drag-and-drop vai direto do browser para o Supabase Storage via signed URL. O servidor não está no caminho do upload.
- Groq Whisper Large v3 transcreve com timestamps por segmento.
- Tipo MIME e limite de 100 MB aplicados tanto na server action quanto na bucket policy.
- O writer recebe os segmentos da transcrição marcados com índices entre colchetes; as citações referenciam esses índices.
- A UI em split-view rola até o segmento citado ao passar o mouse.

### B2B multi-tenant

Workspaces, roles (`OWNER`, `ADMIN`, `MEMBER`), convites, API keys, log de auditoria. Três camadas de isolamento — escopo por linha no Postgres, guards na camada de aplicação em cada server action e bucket policies do Supabase Storage com chave por workspace ID.

## Status

Em desenvolvimento ativo. O pipeline de agentes e a tenancy de workspaces estão funcionando; o polimento da UI e o demo ao vivo estão em andamento.
