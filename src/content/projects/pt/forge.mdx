## O que é

Um estudo, não um produto. O artefato entregue é:

- Uma config de serving no vLLM com continuous batching, KV cache e uma API de streaming compatível com OpenAI.
- Uma receita de quantização AWQ-INT4.
- Um harness de benchmark medindo throughput, TTFT (p50/p95) e TPOT (p50/p95) sob concorrência realista.
- Uma avaliação de qualidade encapsulando `lm-evaluation-harness` (MMLU, GSM8K, HellaSwag) e reportando o delta entre precisão completa e quantizado.
- Um modelo de custo produzindo $/1M tokens self-hosted vs. APIs comerciais.
- Métricas Prometheus + um dashboard Grafana para a etapa de serving.

## Por que construí

Para demonstrar que consigo servir e otimizar um LLM open-source de ponta a ponta — não apenas chamar uma API hospedada. A pergunta interessante não é "o vLLM consegue rodar Llama 3.1 8B" (sim, a documentação cobre isso) — é "como fica o número defensável de $/1M tokens quando você leva em conta utilização, latência p95 e regressão de qualidade da quantização?" É isso que o Forge mede.

## Como funciona

### Desenvolvimento local em hardware comum

O pipeline completo roda em um MacBook Pro M1 base contra um modelo pequeno (`Qwen/Qwen2.5-0.5B-Instruct`) antes de qualquer aluguel de GPU pago. O build CPU do vLLM, o harness de benchmark, os parsers do modelo de custo e o gerador de gráficos são todos depurados localmente de graça. Só a execução final de reprodução consome minutos de GPU.

### Reprodutibilidade

Metodologia, hardware, SHAs do modelo, versão exata do vLLM e receita AWQ ficam em `docs/methodology.md`. O JSON de saída dos benchmarks é commitado em `results/`. Qualquer pessoa com o doc de metodologia e uma GPU alugada consegue reproduzir os números — esse é o ponto do entregável.

### CI

O GitHub Actions roda Ruff (lint + format), mypy e pytest em todo PR e push para main. Sem jobs de GPU no CI — a reprodução dos benchmarks é uma etapa manual, guiada pela metodologia, em hardware alugado.

## Status

Scaffolding. A camada de serving, a receita de quantização, o harness de benchmark e o modelo de custo são entregues nas fases 1–8 do plano de implementação; o lead de impacto, a execução completa dos benchmarks e os gráficos chegam na fase 9.
